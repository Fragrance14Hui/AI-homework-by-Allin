# 图论与多智能体系统一致性 学习笔记

# 一、开篇概念

**多智能体系统（MAS）**：简单说就是一群能互相通信、配合干活的“小个体”，比如无人机编队、成群机器人、传感器网络，每个个体就是一个**智能体**。

**一致性问题**：让这群智能体不用中央指挥，只靠邻居之间传信息，最后大家的状态（比如位置、速度、数值）变得**完全一样**，这就是“达成一致”。

**图论的作用**：用来画智能体之间的“通信关系网”，谁和谁能说话、信息怎么传，全靠图论描述，是分析一致性的**基础数学工具**。

---

# 二、图论基础

## 2.1 图的基本概念

一个图记作 **G=(V, E)**，就像一张社交关系图：

- **V（顶点集）**：每个顶点 = 1个智能体，假设有n个智能体，就有n个顶点

- **E（边集）**：每条边 = 2个智能体之间能通信

- 无向边：双向通信，A能找B，B也能找A（最常用）

- 有向边：单向通信，只能A传给B，B不能回传

## 2.2 两个核心矩阵

### 1. 邻接矩阵A

用来记录“谁和谁有通信”的方阵，大小n×n（n=智能体个数）：

- 如果智能体i和j能通信： $a_{ij} = 1$ （简单场景权重取1）

- 如果不能通信： $a_{ij} = 0$ 

- 自己不和自己连： $a_{ii}=0$ （无自环）

### 2. 拉普拉斯矩阵L（重点）

公式：**L = D - A**，是分析一致性的**核心矩阵**

- D：度矩阵（对角矩阵），对角线上的数 = 每个智能体的邻居个数

- 简单性质：L的每一行加起来等于0，0一定是它的特征值

## 2.3 关键：连通性

**连通 = 能达成一致；不连通 = 永远一致不了**

- 无向连通图：任意两个智能体之间，能通过中间节点连上

- 有向图：要有“生成树”（有一个领头节点，所有人都能收到它的信息）

# 三、一致性基础

## 3.1 最简单的智能体模型

只考虑一阶动态（新手入门专用），单个智能体状态变化：

 $\dot{x}_i(t) = u_i(t)$ 

-  $x_i(t)$ ：第i个智能体的状态（比如位置、数值）

-  $u_i(t)$ ：控制指令（只看邻居信息，不用全局指挥）

## 3.2 什么是达成一致？

时间足够长以后，**所有智能体的状态都相等**，数学写法：

 $\lim_{t \to \infty} x_i(t) = x_j(t)，\quad 对所有i,j$ 

## 3.3 基础一致性协议

每个智能体只跟邻居“对齐误差”，控制指令这么写：

 $u_i(t) = \sum_{j∈邻居} (x_j(t) - x_i(t))$ 

合并成全局公式（用拉普拉斯矩阵）：

 $\dot{\mathbf{x}}(t) = -L\mathbf{x}(t)$ 

---

# 四、收敛结论

**只要通信图是连通的，这个基础协议一定能让所有智能体达成一致！**

- 一致结果：所有智能体最终会收敛到**初始状态的平均值**

- 收敛快慢：通信连得越密，收敛速度越快

# 五、易错点

- ❌ 误区：不是所有拓扑都能一致，**不连通的图绝对收敛不了**

- ❌ 误区：拉普拉斯矩阵L不是随便写的，必须满足L=D-A

- ✅ 重点：先画通信图→写邻接矩阵A→算度矩阵D→求拉普拉斯矩阵L→套一致性公式

# 六、小结

用**图**画智能体通信网，用**拉普拉斯矩阵L**建模，靠**邻居误差对齐**，只要拓扑连通，多智能体就能实现分布式一致。
